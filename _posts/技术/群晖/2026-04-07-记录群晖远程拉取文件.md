# 1.引言
记录本文的主要目的是：因为在群晖上搭建了ELK，现在需要从远程服务器上定时拉取日志做日志分析。
主要方式是使用定时任务以及脚本进行批量拉取与解压。群晖系统基于`ubuntu 20.04`进行构建，但是没有`apt`、`sshpass`命令
所以可以使用`密钥连接`的方式或者安装`sshpass`

# 2.免密连接服务器
## 2.1 密钥连接步骤
### 公私钥生成
```
ssh-keygen -t ed25519 -C "synology-log-collector"
```
生成密钥，文件地址`~/.ssh/`

### 复制公钥
生成的密钥有两个文件`id_ed25519`和`id_25519.pub`，前面的是私钥，后面的是公钥
复制
```bash
cat ~/.ssh/id_ed25519.pub | grep synology-log-collector | ssh user@remote-server "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```
通过这个方式将公钥放在远程服务器上

### 远程连接
将密钥文件设置好后，可以使用`ssh`命令直接连接，不需要输入密码
第一次连接提示
```
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@         WARNING: UNPROTECTED PRIVATE KEY FILE!          @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
Permissions 0777 for '~/.ssh/id_ed25519' are too open.
It is required that your private key files are NOT accessible by others.
This private key will be ignored.
Load key "~/.ssh/id_ed25519": bad permissions
```
意思为，私钥的权限太高，需要设置为`600`，再次连接即可成功


## 2.2 sshpass连接
### 下载deb
因为群晖上没有`apt`，通过`dpkg -i`命令进行安装，需要手动下载`sshpass`命令的包
[地址](https://ubuntu.pkgs.org/)
通过命令查看群晖发行版本
```bash
cat /etc/*-release
```
如果没有就随便下载吧（什么时候能找到发行版本再补充方法）

### 安装
```bash
dpkg -i sshpass.deb
```
deb文件提示需要依赖`libc6`，但是`dpkg`不能自动安装依赖，安装时提示`no signature`
```
dpkg -i --force-all sshpass.deb
```
`--force-all`即可安装成功，但是提示没有依赖，尝试了以下可以运行

### 使用
在使用`ssh`和`scp`等命令时，使用`sshpass`进行密码填充
```
sshpass -p "123456" ssh user@remote-server
sshpass -p "123456" scp user@remote-server:/ .
```

# 3.自动化脚本

```bash
#!/bin/bash
USER_NAME=ligen
IP=127.0.0.1
PORT=55537

cd /volume5/docker/program/prod/schola/logs/
env_path=`pwd`
echo -e $env_path
echo -e "downloading logs..."
scp -P $PORT $USER_NAME@$IP:/home/ligen/workspace/yudao/logs/*.gz logs/
echo -e "delete logs..."

ssh -p $PORT $USER_NAME@$IP "rm /home/ligen/workspace/yudao/logs/*.gz"

echo -e "move logs to filebeat..."
cd logs
gzip -d *.gz
for file in *.0; do mv "${file}" "${file}.log"; done

mv *.log /volume5/docker/program/prod/elk/filebeat/business/schola/
```
